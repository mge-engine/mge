Render Context

Render APIs use a context object that combines objects of the render pipeline. A render context is btained from a mge::window, and has a 1:1 relationship with the window.

class render_context

A render context is a context in which objects that are elements of the render pipeline are created - such as shader programs, textures and buffers.

A render context has a swap chain to facilitate presentation of the rendered frames.

Context objects are created based upon a render context. Their life time is bound to the render context, and they are valid for one render context only.

class context_object : public mge::noncopyable

An object that is valid in the context of a render_context like a shader program, a texture, or a command list.

Subclassed by mge::command_list, mge::hardware_buffer, mge::program, mge::shader, mge::swap_chain, mge::texture

Each render context has a swap chain, a series of buffers to stabilize frame rates and enable presenting frames.

class swap_chain : public mge::context_object

A swap chain is a series of virtual frame buffers.

Commonly, a swap chain is used for frame rate stabilization by using double, or even triple buffering.
