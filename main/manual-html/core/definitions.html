Definitions¶

System Identification¶

While MGE strives for portability, it acknowledges it is sometimes necessary to identify the nature of the environment. So, some identification macros are provided, in mge/config.hpp.

Note

While MGE targets currently only Microsoft Visual Studio, some other definitions are provided, however no implementation exist for other operating systems, compilers, and CPU architectures

Compiler¶

MGE_COMPILER_MSVC¶

Defined if the compiler is Microsoft Visual C++.

MGE_COMPILER_GNUC¶

Defined if the compiler is GNU C++.

MGE_COMPILER_MINGW¶

Defined if the compiler is MinGW (in this case also MGE_COMPILER_GNUC is defined).`

MGE_COMPILER_CLANG¶

Defined if the compiler is clang.

Operating System¶

MGE_OS_LINUX¶

Defined if the operating system is Linux.

MGE_OS_WINDOWS¶

Defined if the operating system is Microsoft Windows.

MGE_OS_MACOSX¶

Defined if the operating system is Mac OS X.

MGE_OS_UNIX¶

Defined if the operating system is a Unix-style system, like Linux and Mac OS X.

Attributes¶

Attributes are additional tags on declarations that influence how the compiler interprets them (and what conditions need to be warned on).

Common Attributes¶

MGE_NO_RETURN¶

Attribute that describes a function does not return ([[noreturn]] in C++11).

MGE_NO_INLINE¶

Attribute forcing a function not to be inlined.

Symbol Visiblity¶

The file mge/dllexport.hpp defines some macros used for symbol visiblity.

MGE_DLLEXPORT¶

Macro to declare class or function to be exported from DLL.

MGE_DLLIMPORT¶

Macro to declare class or function to be imported from DLL.

Utilities¶

MGE_FUNCTION_SIGNATURE¶

Expands to the function signature as text.
