Tracing¶

MGE defines a tracing using a system of trace topics having active trace levels and trace sinks configured.

  • A trace level defines whether a trace message is informational or describes a severe error.

  • A trace sink is a mechanism to write trace records to - like a file or the console.

  • A trace topic is a named category of trace, which allows an active level to be set, and trace sinks attached as destination of trace records

  • A trace formatter is a mechanism to create text from a trace record.

  • A trace record is created for each trace entry, trace records capture the trace message and details on its origin.

To use tracing, mge/core/trace.hpp needs to be included.

Severity¶

Trace messages can have a different severities, indicated by a trace level.

Severity

Purpose

ERROR

Severe malfunction message.

WARNING

Potential problem.

INFO

General information message.

DEBUG

Detailed debug information.

FATAL

Fatal message, program will exit.

Trace Topic¶

Trace topics partition trace output into different categories. Trace can be enabled or disabled per category. There exists a pseudo-category for enabling and disabling trace at a global level.

Defining a Trace Topic¶

In a source file, a trace topic must be defined. This is best done using the predefined macro MGE_DEFINE_TRACE.

MGE_DEFINE_TRACE(TOPIC)¶

Define a trace topic.

Parameters

  • TOPIC: name of trace topic

Using a Trace Topic¶

If you just want to use a trace topic for tracing, a forward declaration must be done. This is done using MGE_USE_TRACE, or MGE_USE_IMPORTED_TRACE in case the topic is defined in another dynamic library.

MGE_USE_TRACE(TOPIC)¶

Use an already defined trace topic locally.

Parameters

  • TOPIC: name of trace topic

MGE_USE_IMPORTED_TRACE(TOPIC)¶

Use an already defined trace from another library locally.

Parameters

  • TOPIC: name of trace topic

If you need to inspect a trace topic directly, the following macros can be used:

MGE_TRACE_TOPIC(TOPIC)¶

Access a trace topic.

Parameters

  • TOPIC: name of trace topic

MGE_NS_TRACE_TOPIC(NS, TOPIC)¶

Access a trace topic defined in another namespace.

Parameters

  • NS: namespace

  • TOPIC: name of trace topic

As an example, writing trace into a trace topic TEST looks like this:

MGE_ERROR_TRACE(TEST) << "This is a test of error trace";

Note that a line break isn’t needed at the end.
