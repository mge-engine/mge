Configuration¶

Configuration is achieved by declaration of _parameters_ and later their evaluation at runtime.

Parameter Definition¶

Typed Parameter¶

The parameter interface is further refined into the mge::parameter template class. It allows to retrieve the parameter value as a typed value.

Note

The mge::basic_parameter::from_string() and mge::basic_parameter::to_string() methods are used to convert the parameter value back and forth, this uses a lexical_cast for that purpose.

Configuration Management¶

The configuration is controlled by the class mge::configuration.

static void load(bool allow_missing = true)¶

Parameters

  • allow_missing: succeeds even if no configuration is found

static void store()¶

Store configuration.

static bool loaded()¶

Return whether configuration already has been loaded.

Return

true if already loaded

For loading the configuration, the following process is used:

  1. mge::executable_name() is used to retrieve the base name of the configuration

  2. a valid configuration file is looked up, in the formats supported by boost::property_tree, the first match is used:

    • <executable_name>.json

    • <executable_name>.xml

    • <executable_name>.info

    • <executable_name>.ini

  3. The file is read, and the parameters registered are populated.

If a configuration was loaded, the same file type is used when storing it. If it was not loaded before, a .json file is created.
