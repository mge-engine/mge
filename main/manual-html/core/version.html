Version Tracking

These tools provide utilities for semantic versioning and component tracking. They help developers manage version information and track components throughout the application lifecycle.

Semantic Version

class semantic_version : public boost::less_than_comparable<semantic_version>, public boost::equality_comparable<semantic_version>

A semantic version, comprised of three levels of versioning.

Semantic versioning (https://semver.org) applies certain rules to version number changes:

  • a change in the major version number implies possibility of incompatible API changes

  • a change of the minor version number implies added functionality, but in a backward-compatible manner

  • a change in patch level implies only backward compatible bug fixes
