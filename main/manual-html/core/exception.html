Exception Handling¶

MGE defines a base exception class. This basic class allows to attach additional informations to the exception, such as location information and stack traces.

template<typename Tag, typename Value>
struct tag : public mge::exception::tag_base¶

Exception value tag type.

The tag type is used to attach values to the exception.

tparam Tag

type used to access the value

tparam Value

stored value type

To use it, create a type that will hold the information as a member value, and derive that type from the tag type as follows. 

// A 'foo' value will hold a string attached to an exception.
struct foo : public tag<foo, std::string>
{
    std::string value;
};

Throwing Exceptions¶

The macro MGE_THROW can help in composing an exception.

MGE_THROW(ex)¶

Throw exception instance.

Parameters

  • ex – exception type

If this is an exception translation, i.e. a catched exception is translated into a different error, using MGE_THROW_WITH_CAUSE is more appropriate:

MGE_THROW_WITH_CAUSE(ex, causing_exception)¶

Throw exception and adds a cause.

Parameters

  • ex – exception type

  • causing_exception – exception causing this exception

To attach extra info like the called function, the macro MGE_CALLED_FUNCTION can be used like this:

MGE_THROW(mge::exception)
    << MGE_CALLED_FUNCTION(glGenBuffers)
    << "Cannot generate buffers";
MGE_CALLED_FUNCTION(X)¶

Helper to add called function to exception information.

Parameters

  • X – name of called function This helper is usually used to attach additional information to the exception, e.g. in case of system or foreign API errors. 

    MGE_THROW(mge::exception) << MGE_CALLED_FUNCTION(fopen);
